一、台股爆發前的專業分析 (PDF 原文)

1. 技術面：突破與放量

長期盤整後的首跳： 股價在橫盤整理 3-6 個月後，突然以一根「帶量長紅」或「跳空缺口」突破所有均線。
成交量異常放大： 在股價還沒大幅拉升前，成交量會先放大到平日的 3-5 倍，代表大戶正在「進場吃貨」。

2. 籌碼面：法人先行

投信連買： 台灣市場有個特色，當「投信」開始連續 3-5 天淨買入某隻中小型股時，往往是爆發的前奏。
融資與借券： 融資餘額溫和增加但股價穩定上升，顯示資券同作的「軋空」動能正在醞釀。

3. 基本面：營收與消息

月營收 YoY/MoM 轉強： 每月 10 號前公布的營收，若出現連三個月成長，且創下近期新高。
「含金量」高的消息： 例如公司宣布進入某大廠 (如 Apple 或 Nvidia) 的新供應鏈名單，這類訊息在市場初期通常被低估。

四、給您的風險提示

這種一個月成長 50% 的股票通常伴隨著極高的波動性。一旦趨勢反轉，跌落的速度也會非常快。建議觀察「五日線」作為移動停利點：只要股價不破五日線，強勢行情可能持續；一旦跌破，往往是獲利了結的信號。

三、核心算法與 API 開發參考 (PDF 與對話實錄)

核心監控邏輯 (The Detection Engine):
請開發 Agent 在 src/lib/engine.ts 中實現以下加權評分函數：

$${Score = (V_{ratio} \times 40) + (MA_{constrict} \times 30) + (Inst_{flow} \times 30)}$$

具體判斷指標 (算法核心)：

📊 量能激增 ($V_{ratio}$)

條件：

  • 當日成交量 > 過去 20 日平均成交量 3.5 倍。
  • 且成交量連續 3 天遞增。
📉 均線擠壓 ($MA_{constrict}$)

條件：

  • 計算 $ABS(MA5 - MA20) / MA20 < 0.02$ (代表均線高度糾結)。
  • 當日收盤價突破糾結區且漲幅 > 3%。
💰 籌碼集中 ($Inst_{flow}$)

條件：

  • 投信 (Investment Trust) 連續 3 日淨買超。
  • 或外資與投信「對作轉同步」。

需要串接的 FinMind API 端點 (Endpoints)：

  • TaiwanStockPrice: 獲取日 K 線, 計算 $V_{ratio}$ 與均線。
  • TaiwanStockHoldingSharesPer: 重點抓取 Investment_Trust (投信) 的持股增減。
  • TaiwanStockInfo: 用於對照股票代號與名稱。
  • TaiwanStockMonthRevenue: 用於過濾具備基本面支撐的標的。

APP 終端開發規範 (Web / PWA)

1. 平台架構： Next.js 14+ (App Router), TypeScript, Tailwind CSS.
2. 部署目標： GitHub 託管代碼，自動部署至 Netlify。
3. PWA 優化： 必須配置 manifest.json 與 apple-mobile-web-app-capable，支持 iOS「加入主畫面」。
4. 自適應設計： 豎屏顯示清單，橫屏 (Landscape) 自動全屏顯示 TradingView K 線圖。
