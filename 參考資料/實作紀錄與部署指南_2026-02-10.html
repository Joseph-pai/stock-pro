1. 系統進階方案：導入 Redis 全局緩存實施計劃

本計劃旨在探討如何透過導入 Redis 將系統從「客戶端分段暫存」升級為「服務端全局緩存」，以極大化數據讀取速度並降低外部 API 負擔。

核心目標

  • 消除網路延時：將數據抓取延遲從 500ms+ 降低至 20ms 以內。
  • 節省 API 配額：減少對 FinMind、TWSE 等外部資源的重複請求。
  • 全局共享：實現不同設備（手機、電腦）間的緩存共享。
  • 預熱機制：開盤前自動準備數據，實現秒級掃描。

數據類型 Redis Key 範例 TTL (過期時間) 說明

穩定性：即使外部 API 暫時當機，只要 Redis 內有緩存，系統依然能提供正常的數據服務。

2. Netlify 部署調整指南 (針對 Redis 緩存)

由於 Netlify 是基於 Serverless Functions 的架構，部署時需要注意以下關鍵點以確保 Redis 緩存正常運作：

1. 外部 Redis 服務 (必要)

Netlify 的伺服器本身不提供 Redis。您需要一個外部的 Redis 實例：

  • 建議服務：Upstash (專為 Serverless 設計)。
  • 優點：支持透過 HTTP 調用且具備極佳的連結管理機制。

2. Vercel KV 操作步驟 (獲取 Redis URL)

  1. 登入 Vercel 並連結 GitHub 倉庫。
  2. 點擊頂部的 Storage 分頁，建立一個 KV Database。
  3. 選擇區域 (例如 Washington, D.C. (us-east-1))。
  4. 在 KV 儀表板中，複製 REDIS_URL 連結字串。

3. Netlify 環境變數設置

在 Netlify 的 Site Settings > Build & deploy > Environment variables 中添加：

變數名稱 值
REDIS_URL 填入外部 Redis 的連結字串 (格式: redis://default:password@host:port)
NEXT_PUBLIC_FINMIND_TOKEN 您的 FinMind API Key
靜態預渲染限制：由於 API 現在依賴數據庫連結，如果您在編譯期 (npm run build) 就需要抓取數據，請確保編譯環境也能訪問到 Redis。
