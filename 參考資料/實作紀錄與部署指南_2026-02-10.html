1. 系統進階方案：導入 Redis 全局緩存實施計劃

本計劃旨在探討如何透過導入 Redis（遠端字典服務）將系統從「客戶端分段暫存」升級為「服務端全局緩存」，以極大化數據讀取速度並降低外部 API 負擔。

1. 核心目標

  • 消除網路延時：將數據抓取延遲從 500ms+ 降低至 20ms 以內。
  • 節省 API 配額：減少對 FinMind、TWSE 等外部資源的重複請求。
  • 全局共享：實現不同設備（手機、電腦）間的緩存共享。
  • 預熱機制：開盤前自動準備數據，實現秒級掃描。

2. 緩存鍵 (Key) 設計架構

為確保數據可維護性，採用層級化命名空間：

數據類型 Redis Key 範例 TTL (過期時間) 說明
市場快照 tsbs:snap:{mkt}:{sector} 1 小時 存放 TWSE/TPEX 類股快照
歷史 K 線 tsbs:hist:{symbol} 24 小時 存放個股歷史交易數據
AI 分析 tsbs:ai:{symbol}:{date} 12 小時 存放昂貴的 AI 深度分析結果
產業映射 tsbs:map:industry 7 天 變動頻率極低的映射表

3. 資料流邏輯 (Read-Aside Pattern)

graph TD A[API 請求檢索] --> B{檢查 Redis 緩存} B -- 命中 (Hit) --> C[回傳 Redis 數據] B -- 未命中 (Miss) --> D[調用外部 API 抓取] D --> E[寫入 Redis 並設 TTL] E --> F[回傳 最新數據]

技術實施點：

  1. 緩存穿透保護：針對不存在的代號（如 0000），在 Redis 存入空值短暫過期，防止頻繁請求外部伺服器。
  2. 自動失效控制：利用 Redis EXPIRE 屬性，確保歷史數據在收盤後自動更新。

4. 進階優化：預熱機制 (Cache Warming)

透過後端 Cron Job 執行數據預熱：

  • 08:30 (開盤前)：自動跑一次 Market Snapshot 抓取，填滿 Redis。
  • 14:00 (收盤後)：自動執行全市場 Deep Analysis，緩存當日的籌碼與指標分析。

5. 實施效益評價

性能提升：對於使用者來說，掃描 200 支股票的等待時間將從「感覺有在跑」變成「瞬間完成」。

穩定性：即使外部 API 暫時當機，只要 Redis 內有緩存，系統依然能提供正常的數據服務，提升健壯性。

2. Netlify 部署調整指南 (針對 Redis 緩存)

由於 Netlify 是基於 **Serverless Functions** (無伺服器函數) 的架構，部署時需要注意以下幾點以確保 Redis 緩存正常運作：

1. 外部 Redis 服務 (必要)

Netlify 的伺服器本身不提供 Redis。您需要一個外部的 Redis 實例：

  • 建議服務：[Upstash](https://upstash.com/)。
  • 原因：Upstash 是專為 Serverless 設計的，支持透過 HTTP 調用且具備極佳的連結管理機制，非常適合 Netlify 使用。
  • 替代方案：Redis Cloud, Aiven 或 AWS ElastiCache。

2. Vercel KV 詳細操作步驟 (獲取 Redis URL)

如果你選擇使用 Vercel KV 作為後端：

  1. 登入 Vercel：訪問 [vercel.com](https://vercel.com/)。
  2. 建立專案：
    • 點擊 Add New... > Project。
    • 連結你的 GitHub 倉庫並點擊 Import。
    • (可以不用管部署是否成功，我們只需要數據庫)。
  3. 啟用 KV 存儲：
    • 進入專案後，點擊頂部的 Storage 分頁。
    • 點擊 Create Database。
    • 選擇 KV 並點擊 Continue。
    • 接受條款並選擇一個接近你用戶的區域 (例如 Washington, D.C. (us-east-1))。
    • 點擊 Create。
  4. 獲取連結字串：
    • 在 KV 儀表板中，找到 .env.local 區塊。
    • 找到以 REDIS_URL= 或 KV_URL= 開頭的那一行。
    • 點擊複製複按鈕 獲取整個連結 (格式如 redis://default:xxx@xxx.upstash.io:6379)。

3. Netlify 環境變數設置

在 Netlify 的 Site Settings > Build & deploy > Environment variables 中，必須添加以下變數：

  • REDIS_URL：填入外部 Redis 的連結字串 (格式通常為 redis://default:password@host:port)。
  • 如有其他 API Key (如 FinMind)，也請一併在此設置。

3. 代碼優化建議 (ioredis)

目前的 src/lib/redis.ts 雖然支持單例模式，但在 Serverless 環境中，每個新的函數實例還是會建立一次連接。

注意：如果您使用的是有「連結數限制」的免費版 Redis，請監控連結數。

進階建議：若未來流量變大，建議將 ioredis 更換為 @upstash/redis (基於 HTTP)，以完全避免 TCP 連結池在 Serverless 中造成的管理問題。

4. 靜態預渲染 (ISR/SSG) 限制

由於 API 現在依賴數據庫連結，如果您在編譯期 (npm run build) 就需要抓取數據，請確保編譯環境也能訪問到 Redis，否則編譯可能會出錯。

總結步驟：
  1. 註冊 Upstash 並建立一個免費的 Redis 數據庫。
  2. 複製連結字串 (Redis URL)。
  3. 在 Netlify 後台設置環境變數。
  4. 重新部署專案。
