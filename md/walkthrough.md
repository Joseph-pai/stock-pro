# 深度優化與算法升級成果報告 (2026-03-04)

## 優化概覽
本次升級聚焦於核心選股邏輯的「精準度」與系統在資料缺失時的「透明度」。

---

## 核心升級項目

### 1. 算法精準化 (Algorithm Precision) 🎯
- **多頭排列 (Bullish Arrangement)**：
  - 新增 `MA5 > MA20` 檢查。
  - 對於非多頭排列股進行 `0.5x` 分數扣減，多頭股給予 `1.2x` 加成，有效過濾弱勢股。
- **平滑分數衰減 (Smooth Decay)**：
  - 取代原本的 10% 斷崖歸零，改用 **指數衰減模型 (Exponential Decay)**。均線剛開始擴散時分數依然存在，讓「強勢快突破」但均線略開的個股不被遺漏。
- **量能基線校正**：
  - 統一將 `vRatio` 基線設定為 **45 日均量**，避免短期波動誤導「爆量」判斷。

### 2. 籌碼與融資模型相對化 (Contextualization) 💎
- **籌碼佔比權重**：
  - 引入「投信買張 / 總成交量」佔比。若佔比超過 **1%** 即視為極強勢主力布局，給予額外加分。
- **相對融資門檻**：
  - 將原本硬編碼的 500 張門檻優化。現在考量到融資與成交量的相對性，提升了大盤股與小型股的適配性。

### 3. 小型爆發股發現 (Discovery) 🚀
- **預篩選優化**：
  - 調降 `scanMarket` 的成交量門檻至 **1000 股**。
  - 按「成交量 * 爆量因子」排序，確保原本成交量低、但今日突然爆發數倍的小型黑馬能進入前 200 名分析清單。

### 4. 系統健壯性 (Robustness) 🛠️
- **警告標記系統 (Warnings)**：
  - 在 `AnalysisResult` 中加入 `warnings` 欄位。
  - 當 FinMind 資料缺失或回退到 Exchange 時，在報告中顯示「資料不全」警告，防止用戶誤以為評分低，實則是 API 故障。
